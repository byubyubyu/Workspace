# 3Dキャラ生成パイプライン（AI画像 → ローポリ3D → ボーン → Unity）

確立日: 2026-06-11（第1号: Soldier_Female_01）
仮プリミティブ（兵士・市民・モンスター等）をAI生成のローポリ3Dモデルへ置き換えるための手順書。

---

## 必要環境

| ツール | 場所 / 起動 | 役割 |
|---|---|---|
| ComfyUI-3D | `E:\ai\ComfyUI-3D`（`Launch.bat`・ポート8188） | 参照画像生成 + Hunyuan3D 2.1で画像→3D |
| Blender + MCPアドオン | Blender起動 → Nキー → BlenderMCPタブ → Connect | クリーンアップ・ボーン・ウェイト・FBX出力 |
| Unity 6 + Funplay MCP | 本プロジェクト | 取り込み・Humanoid設定・検証 |

## 手順

### 1. 参照画像の生成（ComfyUI / `anima1.json` ベース）

- 768×768・waiANIMA・BiRefNet背景除去 → **アルファ透過PNG**
- プロンプト必須要素:
  - **Aポーズ**（腕は水平〜45度下げ。真横Tポーズより肩の変形が良い）
  - **開脚**（`legs apart, wide stance, feet shoulder-width apart`）←最重要
  - `front view, full body, head to toe, simple background, white background`
  - 地面・影・武器を **ネガティブに明記**（`ground, floor, shadow, weapon, sword, ...`）
- 出力は `output/` に出る → `input/` へコピーして3D工程の入力にする

> **なぜ開脚必須か**: 脚が閉じていると2,000面削減の段階で膝・靴の間にポリゴンの橋が
> できて融合する。ウェイトでは直せない。脚は生成後にボーンで閉じる（手順4）。

### 2. Hunyuan3D 2.1 で 3D化（`Full_Workflow.json` ベース）

- 設定値:
  - `NumberOfFaces` = **2000**（兵士サイズはこれで十分。粗ければ3000〜5000）
  - `Hy3D21LoadImageWithTransparency.image` = 手順1のPNG
  - `OutputMeshName` = `Unity/<モデル名>`
- API実行時の罠:
  - SetNode/GetNode（仮想ノード）は `/prompt` で400 → 直結に書き換える
  - `Hy3DMeshGenerator.attention_mode` は `sdpa` にする（保存値`randomize`は不正）
- 出力: `output/Unity/<モデル名>.glb`（テクスチャ焼き込み済み・約1MB）
  - `output/Unity/` は `C:\game\dream\Assets\...` への**ジャンクション**（旧プロジェクト）。
    本プロジェクトで使う分はBlender経由でFBX化するので触らなくてよい
- 所要: 約5分（モデルロード込みだと7〜8分）

### 3. Blender クリーンアップ + ボーン + 自動ウェイト

1. GLBインポート後、**必ず `remove_doubles(threshold=0.0001)`**
   - UVシーム由来の重複頂点（約1,400個）が残っているとボーンヒートが**全ボーン失敗**する
   - UVはループ単位保存なのでテクスチャは壊れない
2. マテリアル名を **`SoldierMat` 等の固定名に毎回リネーム**
   - 再インポートで `.001` が付くとUnity側のマテリアルリマップが外れて**真っ白**になる
3. Unity Humanoid互換 **17ボーン** を頂点分布から配置:
   - Hips / Spine / Chest / Neck / Head ＋ Shoulder・UpperArm・Forearm・Hand（L/R） ＋ Thigh・Shin・Foot（L/R）
   - 腕の傾き: `|x|>0.35 かつ z>55%H` の頂点を直線フィット
   - 脚の位置: すね高さ(10-20%H)・腿上端(42-48%H)のx平均（左右別・非対称OK）
   - この命名でUnityのHumanoid自動マッピングが通る（Rigifyは生成リグが汚いので不採用）
4. `parent_set(type='ARMATURE_AUTO')` → ウェイト後処理:
   - **左右にじみ除去**: 脚系（Thigh/Shin/Foot）は反対側（x符号違い）の頂点からウェイト削除 → 正規化
   - **頭部剛体化**: z>0.70は Head=1.0 固定、0.60〜0.70はHead/Neckブレンド（顔のグニグニ変形防止）
5. ポーズ修正（開脚→自然な立ち姿）:
   - 腿ボーンをワールドY軸回りに内側へ回転（足首間隔 約0.3mに）
   - メッシュへ焼き込み（Armatureモディファイア複製→片方適用）→ `Apply Pose as Rest Pose`
6. FBXエクスポート:
   - `add_leaf_bones=False, apply_scale_options='FBX_SCALE_ALL', path_mode='COPY', embed_textures=True`
   - テクスチャは別途PNGでも書き出す（Unityマテリアルはこちらを参照）

### 4. Unity 取り込み

1. FBXを `Assets/Models/<モデル名>/` へ、テクスチャを `Textures/` サブフォルダへ
2. ModelImporter: `animationType = Humanoid` → Avatar `valid/human` を確認
3. URP Litマテリアル新規作成（_BaseMap=albedo）→ ImporterのMaterial Remapで
   ソース名 **`SoldierMat`** → 作成マテリアル に張り替え
4. 検証: シーン配置 → ボーン回転 or テスト用スクリプトで変形確認

## アニメーション（Mixamoリターゲット・確立済み 2026-06-11）

1. Mixamoで**アップロード不要**（デフォルトキャラのまま）アニメを選び、
   歩き/走りは **In Place** にチェック → FBX Binary / **Without Skin** でDL
2. `Assets/Models/Animations/Mixamo/` に配置 → ModelImporterで `animationType=Humanoid`、
   クリップを `loopTime=true`
3. Animator Controller＝Speed float の1Dブレンドツリー（Idle→Walk→Run）
   - **しきい値は実測値で**（定義値ではなくスキル等の補正込み。実測: 歩き6.0 / 走り10.0）
4. `LocomotionAnimatorDriver`（Scripts/Common/）をビジュアルに付ける
   （親のCharacterController水平速度をAnimatorの"Speed"へ。applyRootMotion=false）
5. **Humanoidの接地仕様**: Animatorルート＝接地点。ビジュアルはPlayerルート（体中心）
   から **localY=-1.0**（CC底面）に置く。+オフセットすると全身が浮く

### アニメ調整の知見（2026-06-11追記）

- **Mixamoクリップ自体に傾き癖がある**（モーキャプ由来）。素のMixamoスケルトンで再生しても
  Standard Runは平均-6°左に傾く。導入前に PlayableGraph でクリップをサンプルし、
  「上体lean」「骨盤roll」を数値測定して選別する（同名の別テイクが複数あるので複数DLして比較）
  - 上体leanが大きい → 走りの見た目が斜めになる
  - 骨盤rollが大きい → 低い側の足が引っかかる・ぐにゃる
- 採用: 走り=Standard Run（骨盤がきれい）＋ `PostureCorrector`（Common/）で上体だけ
  LateUpdateロール補正（-9.5°・速度6→10でフェードイン）。歩き=Walking別テイク（timeScale 1.7）
- **足滑り（スケート）**: ゲーム移動速度がクリップ想定よりずっと速いので、
  ブレンドツリーの timeScale で歩調を地面速度に合わせる
- **閉じ回転の罠**: 腿をY軸回転で閉じると足ボーンに同角のロールが乗る
  → ベイク後に Foot.L/R の tail をワールド正面に直し roll=0 にする（必須）
- Foot IK は試した結果オフ（体の傾きと組み合わさると低い側の足がゴム化）
- 脚の開き幅はレストポーズが基準になる。現在の足首間隔=±0.11

### 攻撃モーション（2026-06-11確立）

- Animator構成: **上半身レイヤー**（AvatarMask=背骨・頭・両腕のみ・Override・weight1）に
  攻撃ステートを置く → 移動しながら攻撃しても脚はロコモーション継続
- `AttackAnimatorMotion`（Common/）: 既存 `Attack` のフェーズを監視（読むだけ・戦闘コード無変更）。
  攻撃開始でCrossFade再生・終了で空ステートへ復帰。
  **クリップ再生速度 = clipLength / AttackMove.TotalTime** で当たり判定とアニメが自動同期
- 技→アニメの振り分けは `AttackMove.motionId` ↔ ステート名のマッピング（インスペクタ設定・
  未設定はfallbackState）。導入済みステート: Attack_OneHand / Attack_TwoHand
- 攻撃クリップのインポート: Humanoid・**loopTime=false**・クリップ名をステート名にリネーム
  （AttackAnimatorMotionが名前でクリップ長を引くため）

### 武器の持たせ方・抜刀演出（2026-06-12確立）

- **武器プレハブ規約: 原点=グリップ（握り点）・刃=長軸+Z（Blenderでは+Z上）**。
  これを守れば HandPoint のオフセット無調整で武器を差し替えられる
- 持たせ方＝`HandBoneFollower`（Common/）: 手ボーンに HandPoint を追従させ、
  既存の PlayerHandState がそこへ武器ビューを出す（戦闘コードの構造は不変）
  - オフセットはボーンローカル値。**キャラごとに1回較正が必要**（実測スニペットで自動算出）
  - 較正法: 武器の長軸ワールド方向を測り `FromToRotation` で目標方向（前方+35度上等）へ合成
- 背負い表示＝BackPoint（Spineボーン追従の2個目のHandBoneFollower）＋PlayerHandState.RefreshBackView
  - **注意: このモデルはHumanoidのChest/UpperChestが未マップ（Spineのみ）**
- 抜刀/納刀の演出（PlayerHandState改修済み）:
  - 手ぶら=背中に表示 / 抜刀中=手に持つが `drawViewEuler` で刃先上向き / 構え=通常向き /
    納刀=モーション完了まで遅延破棄（`sheatheViewLinger`・刃先上向き）

### Animator実装の罠（2026-06-12・重要）

- **`Animator.CrossFade` の遷移時間は「現ステート長に対する割合」**。Idle16秒からの0.2は3.3秒かかる。
  スクリプトからは必ず `CrossFadeInFixedTime`（秒指定）を使う
- **静止ポーズクリップは1秒尺で作る**。1フレーム尺(0.033s)だと30Hzでループ＝ルート再評価でブルブル振動する
- **ブレンドツリーの子は正規化時間で同期**。子1つのtimeScaleを極端に下げると全子（歩き含む）が引きずられ止まる
- 構えポーズの調整＝ポーズ.animのカーブ直接編集で可能（マッスル名は "Spine Front-Back" 等・
  RootT.y=腰の高さ・RootQ=腰の回転・Foot Up-Down は負=つま先上げ）。
  ただし**ランタイムのPlayableGraphサンプリングにはカーブ編集が反映されない**ことがある（SaveAssets後にAnimator経由でのみ反映）→ 調整は「編集→ユーザー目視」のループで行う
- Foot IK は接地ズレ補正に使えるが、ポーズと地面の乖離が大きいと膝がパキパキ動く。
  乖離の根本（RootT.y）を直してIKはオフが安定

## 既知の制限・次の課題

- ブレンドしきい値は実測固定。移動速度バフで変わるため、本実装では
  「実効最高速度で正規化した0〜1」をAnimatorに渡す方式へ（PlayerMovement小改修が必要）
- 被弾・死亡・回避モーション未導入（Mixamoから追加DLで同じ手順・導入前に傾き測定）
- 両手技のmotionId="TwoHand"設定（AttackMove SO側）はまだ＝現状全技が片手斬り
- PostureCorrectorの補正量・脚の開き幅は暫定値（ユーザー調整中）
- スカートにボーンなし（腿ウェイトで追従。激しい動きで貫通の可能性）
- MR（metallic/roughness）テクスチャは未接続（albedoのみ。質感が必要になったら接続）
- ComfyUIカスタムノード `ComfyUI-Hunyuan3d-2-1` に**ローカルパッチ適用済み**
  （中間ファイルをtemp→`hy3d_work/`へ・サブフォルダ自動作成。ノード更新で消えるので注意）
